float NovelRT::Maths::GeoVector4F::Distance ( GeoVector4F  other)
inlinenoexcept

Calculates the distance between this GeoVector4F and another GeoVector4F.

The calculation used to obtain the distance can be seen in this example:

\begin{align*} \vec{v}_{1} &= \begin{pmatrix} 3\\ 10\\ 5\\ 3 \end{pmatrix}\\ \vec{v}_{2} &= \begin{pmatrix} 10\\ 12\\ 15\\ 7 \end{pmatrix}\\ \vec{\Delta v} &= \vec{v}_{2} - \vec{v}_{1}\\ &=\begin{pmatrix} 10 - 6\\ 12 - 10\\ 15 - 5\\ 7 - 3 \end{pmatrix} \\ &= \begin{pmatrix} 7\\ 2\\ 10\\ 4 \end{pmatrix}\\ \left \| \vec{\Delta v} \right \| = \sqrt{{\Delta v}_x^2 + {\Delta v}_y^2 + {\Delta v}_z^2 + {\Delta v}_w^2}\\ &= \sqrt{7^2 + 2^2 + 10^2 + 4^2} \rightarrow \sqrt{49 + 4 + 100 + 16}\\ &= \sqrt{169}\\ &= 13 \end{align*}

This is a pure method. Calling this without using the result has no effect and can introduce overhead for calling the method.

GeoVector4F NovelRT::Maths::GeoVector4F::GetNormalised ( ) const
inlinenoexcept

Constructs a unit GeoVector4F with a magnitude of one by normalising this vector.

To normalise a vector you need to divide its components by its magnitude. As a result, a vector with the resulting components will have a magnitude of 1, also known as a unit vector.

\begin{align*} \vec{v}&= \begin{pmatrix} 7\\ 2\\ 10\\ 4 \end{pmatrix}\\ \left \| \vec{v} \right \| &= \sqrt{v_{x}^{2} + v_{y}^{2} + v_{z}^{2} + v_{w}^{2}} = \sqrt{7^{2} + 2^{2} + 10^{2} + 4^{2}} \rightarrow \sqrt{49 + 4 + 100 + 16} = \sqrt{169} = 13\\ \hat{v}&=\frac{\vec{v}}{\left \| \vec{v} \right \|} = \frac{\bigl(\begin{smallmatrix} 7\\ 2\\ 10\\ 4 \end{smallmatrix}\bigr)}{13} = \begin{pmatrix} \tfrac{7}{13}\\ \tfrac{2}{13}\\ \tfrac{10}{13}\\ \tfrac{4}{13} \end{pmatrix}\\ \left \| \hat{v} \right \| &= \sqrt{\tfrac{7}{13}^{2} + \tfrac{2}{13}^{2} + \tfrac{10}{13}^{2} + \tfrac{4}{13}^{2}} = \sqrt{\tfrac{49}{169} + \tfrac{4}{169} + \tfrac{100}{169} + \tfrac{16}{169}} \rightarrow \sqrt{\tfrac{169}{169}} \rightarrow \sqrt{1} = 1 \end{align*}

This is a pure method. Calling this without using the result has no effect and can introduce overhead for calling the method.

Calculates the squared distance between this GeoVector4F and another GeoVector4F.

The calculation used to obtain the squared distance can be seen in this example:

\begin{align*} \vec{v}_{1} &= \begin{pmatrix} 3\\ 10\\ 5\\ 3 \end{pmatrix}\\ \vec{v}_{2} &= \begin{pmatrix} 10\\ 12\\ 15\\ 7 \end{pmatrix}\\ \vec{\Delta v} &= \vec{v}_{2} - \vec{v}_{1}\\ &=\begin{pmatrix} 10 - 6\\ 12 - 10\\ 15 - 5\\ 7 - 3 \end{pmatrix} \\ &= \begin{pmatrix} 7\\ 2\\ 10\\ 4 \end{pmatrix}\\ \left \| \vec{\Delta v} \right \|^2 = {\Delta v}_x^2 + {\Delta v}_y^2 + {\Delta v}_z^2 + {\Delta v}_w^2\\ &= 7^2 + 2^2 + 10^2 + 4^2 \rightarrow 49 + 4 + 100 + 16\\ &= 169 \end{align*}

This is a pure method. Calling this without using the result has no effect and can introduce overhead for calling the method.

The squared distance between this GeoVector4F and another GeoVector4F.
