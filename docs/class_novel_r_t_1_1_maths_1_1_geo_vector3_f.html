GeoVector3F NovelRT::Maths::GeoVector3F::Cross ( GeoVector3F  other)
inlinenoexcept

Calculates the cross product of this GeoVector3F with another GeoVector3F.

The magnitude of the resulting vector is equal to the area of a parallelogram with this GeoVector3F and the other GeoVector3F as sides.

To calculate the cross product of two vectors you take the following steps:

\begin{align*} \vec{a} &= \begin{pmatrix} 1\\ 5\\ 7 \end{pmatrix}\\ \vec{b} &= \begin{pmatrix} 1\\ 2\\ 3 \end{pmatrix}\\ \vec{c} &= \begin{pmatrix} \vec{a}_{y}\vec{b}_{z} - \vec{a}_{z}\vec{b}_{y}\\ \vec{a}_{z}\vec{b}_{x} - \vec{a}_{x}\vec{b}_{z}\\ \vec{a}_{x}\vec{b}_{y} - \vec{a}_{y}\vec{b}_{x} \end{pmatrix}\\ &= \begin{pmatrix} 5 \times 3 - 7 \times 2\\ 7 \times 1 - 1 \times 3\\ 1 \times 2 - 5 \times 1 \end{pmatrix} \rightarrow \begin{pmatrix} 15 - 14\\ 7 - 3\\ 2 - 5 \end{pmatrix}\\ &= \begin{pmatrix} 1\\ 4\\ -3 \end{pmatrix} \end{align*}

It should be noted that by flipping the left hand side vector with the right hand side vector will result in a cross product of equal length with opposite polarity.

This is a pure method. Calling this without using the result has no effect and can introduce overhead for calling the method.

Calculates the distance between this GeoVector3F and another GeoVector3F.

The calculation used to obtain the distance can be seen in this example:

\begin{align*} \vec{v}_{1} &= \begin{pmatrix} 6\\ 8\\ 3 \end{pmatrix}\\ \vec{v}_{2} &= \begin{pmatrix} 10\\ 12\\ 5 \end{pmatrix}\\ \vec{\Delta v} &= \vec{v}_{2} - \vec{v}_{1}\\ &=\begin{pmatrix} 10 - 6\\ 12 - 8\\ 5 - 2 \end{pmatrix} \\ &= \begin{pmatrix} 4\\ 4\\ 2 \end{pmatrix}\\ \left \| \vec{\Delta v} \right \| &= \sqrt{{\Delta v}_x^2 + {\Delta v}_y^2 + {\Delta v}_z^2}\\ &= \sqrt{4^2 + 4^2 + 2^2} \rightarrow \sqrt{16 + 16 + 4}\\ &= \sqrt{36}\\ &= 6 \end{align*}

This is a pure method. Calling this without using the result has no effect and can introduce overhead for calling the method.

GeoVector3F NovelRT::Maths::GeoVector3F::GetNormalised ( ) const
inlinenoexcept

Constructs a unit GeoVector3F with a magnitude of one by normalising this vector.

To normalise a vector you need to divide its components by its magnitude. As a result, a vector with the resulting components will have a magnitude of 1, also known as a unit vector.

\begin{align*} \vec{v}&= \begin{pmatrix} 2\\ 6\\ 3 \end{pmatrix}\\ \left \| \vec{v} \right \| &= \sqrt{v_{x}^{2} + v_{y}^{2} + v_{z}^{2}} = \sqrt{2^{2} + 6^{2} + 3^{2}} \rightarrow \sqrt{4 + 36 + 9} = \sqrt{49} = 7\\ \hat{v}&=\frac{\vec{v}}{\left \| \vec{v} \right \|} = \frac{\bigl(\begin{smallmatrix} 2\\ 6\\ 3 \end{smallmatrix}\bigr)}{7} = \begin{pmatrix} \tfrac{2}{7}\\ \tfrac{6}{7}\\ \tfrac{3}{7} \end{pmatrix}\\ \left \| \hat{v} \right \| &= \sqrt{\tfrac{2}{7}^{2} + \tfrac{6}{7}^{2} + \tfrac{3}{7}^{2}} = \sqrt{\tfrac{4}{49} + \tfrac{36}{49} + \tfrac{9}{49}} \rightarrow \sqrt{\tfrac{49}{49}} \rightarrow \sqrt{1} = 1 \end{align*}

This is a pure method. Calling this without using the result has no effect and can introduce overhead for calling the method.

Returns
A unit GeoVector3F created from normalising this GeoVector3F.

Calculates the squared distance between this GeoVector3F and another GeoVector3F.

The calculation used to obtain the squared distance can be seen in this example:

\begin{align*} \vec{v}_{1} &= \begin{pmatrix} 6\\ 8\\ 3 \end{pmatrix}\\ \vec{v}_{2} &= \begin{pmatrix} 10\\ 12\\ 5 \end{pmatrix}\\ \vec{\Delta v} &= \vec{v}_{2} - \vec{v}_{1}\\ &=\begin{pmatrix} 10 - 6\\ 12 - 8\\ 5 - 2 \end{pmatrix} \\ &= \begin{pmatrix} 4\\ 4\\ 2 \end{pmatrix}\\ \left \| \vec{\Delta v} \right \|^2 &= {\Delta v}_x^2 + {\Delta v}_y^2 + {\Delta v}_z^2\\ &= 4^2 + 4^2 + 2^2 \rightarrow 16 + 16 + 4\\ &= 36 \end{align*}

This is a pure method. Calling this without using the result has no effect and can introduce overhead for calling the method.
